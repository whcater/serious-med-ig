NameFlagsCard.TypeDescription & Constraints
Location 0..* Location 用于描述一个实体地点的详细信息和位置信息
id 0..1 id 不重复的ID用以识别储存在特定FHIR Server中的Location纪录，通常又称为逻辑性ID。
language 0..1 code 用以表述Location Resource内容的语言。
Binding: CommonLanguages (preferred): 人类语言；鼓励使用CommonLanguages代码表中的代码，但不强制一定要使用此代码表，你也可使用其他代码表的代码或单纯以文字表示。

Additional BindingsPurpose
AllLanguages Max Binding

Example Value: zh-CN
text 0..1 Narrative Location Resource之内容摘要以供人阅读
contained 0..* Resource 内嵌的（contained）、行内的Resources
extension 0..* Extension 扩展字段
modifierExtension 0..* Extension 此扩展字段可能会完全修正或改变其他资料项目的意涵，需特别留意。
identifier 0..* Identifier 地点的唯一识别号
status S 0..1 code active | suspended | inactive
Binding: LocationStatus (required): 地点的状态，指明该地点是否目前正在使用中；应填入所绑定值集中的其中一个代码。

operationalStatus 0..1 Coding 地点的运营状态（通常仅适用于床位 / 房间）
Binding: hl7VS-bedStatus (preferred): 鼓励使用所绑定值集中的代码，但不强制一定要使用此值集，你也可使用其他值集的代码或单纯以文字表示。

name S 1..1 string 人们使用的地点名称
alias 0..* string 这是一份地点过去或现在所使用的其他名称的清单
description 0..1 string 关于地点的额外信息，除了名称之外，还可以提供更多的细节来帮助识别这个地点。
mode 0..1 code instance | kind
Binding: LocationMode (required): 表明为一个地点或是一个地点的类别；应填入所绑定值集中的其中一个代码。

type 0..* CodeableConcept 执行的功能类型
Binding: ServiceDeliveryLocationRoleType (extensible): 应填入所绑定值集中的其中一个代码。

telecom S 0..* ContactPoint 地点的联络细节
system S 0..1 code phone | fax | email | pager | url | sms | other
Binding: ContactPointSystem (required): 应填入所绑定值集中的其中一个代码。

value S 0..1 string 实际的联络方式之细节
use S 0..1 code home | work | temp | old | mobile
Binding: ContactPointUse (required): 此联络方式的用途；应填入所绑定值集中的其中一个代码。

rank 0..1 positiveInt 具体说明偏好的使用顺序（1=最高优先顺序）
period S 0..1 Period 此联络方式的使用效期
address S 0..1 SmAddress 实体地点
physicalType 0..1 CodeableConcept 绝对的地理位置
Binding: LocationType (example): 可参考所绑定值集，但此值集只是针对这个栏位的一个可能值的范例，不预期也不鼓励使用者一定要使用此值集的代码。

position 0..1 BackboneElement 绝对地理位置
longitude 1..1 decimal 以 WGS84 为基准的经度
latitude 1..1 decimal 以 WGS84 为基准的纬度
altitude 0..1 decimal 以 WGS84 为基准的海拔高度
managingOrganization S 0..1 Reference(SmOrganization) 负责地点的提供与维护之机构
partOf 0..1 Reference(SmLocation) 此地点是另一个地点的一部分
hoursOfOperation 0..* BackboneElement 这个地点通常在一个星期的哪些日子/时间开放？
daysOfWeek 0..* code mon | tue | wed | thu | fri | sat | sun
Binding: DaysOfWeek (required): 应填入所绑定值集中的其中一个代码。

allDay 0..1 boolean 此地点全天开放
openingTime 0..1 time 此地点的开放时间
closingTime 0..1 time 此地点的关闭时间
availabilityExceptions 0..1 string 可用性例外情况说明
endpoint 0..* Reference(Endpoint) 提供用于此地点运营服务的技术端点
